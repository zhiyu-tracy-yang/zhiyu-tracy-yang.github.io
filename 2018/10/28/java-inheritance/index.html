Java 继承和重写（待更新）

Java 继承和重写，暂时不更新。

继承 Inheritance

  1. 提高代码的复用性
  2. 类与类之间产生了关系 -> 多态
  3. 只能单继承，只能继承一个
  4. 多层继承：A继承B，B继承C，A可以使用C的方法）

不要为了获取其他类的功能，简化代码而继承
类与类之间要有所属关系才可以继承 (is a)

如果子类中出现非私有的同名成员变量时
子类访问本类的变量用this
子类访问父类的同名变量用super
this代表本类对象的引用
super代表父类对象的引用

重写 Override

当子类出现和父类一模一样的函数时
父类函数被重写

当子类继承父类，父类的功能沿袭到了子类中
但是子类设定的功能不一样
这时，没有必要另写功能，而是使用重写（Override）
保留父类的功能定义，重写功能内容

可以想象成功能要升级，所以需要新的版本

修改原来的代码不方便，初期设计要留意后来代码的扩展性

静态只能覆盖静态
Override要求子类权限大于等于父类权限
权限：public, private, 什么都不写(void show())是默认 - default。最大到最小：public > protected > default > private。
如果子类重写父类时用了private，会报错，因为private（子类）比default（父类）的权限低。子类用default，父类用public的时候也会报错，因为default比public权限低。

如果父类前加了private，可以正常编译，但是子类失去了访问权限

重载Overload和重写Override的区别

重载Overload：只看同名函数的参数列表
重写Override：子父类的方法要一模一样
