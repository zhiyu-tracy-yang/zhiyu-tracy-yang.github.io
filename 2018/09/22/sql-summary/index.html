总结SQL常用语法和知识点

注意事项

  1. 命名规则
    1. 只能使用半角英文字母、数字、下划线(_)作为数据库、表和列的名称
    2. 名称必须以半角英文字母作为开头
    3. 名称不能重复
  2. 所有的列都必须指定数据类型
  3. 表格删除（drop table）后不能恢复
  4. 表格修改（alter table）后不能恢复
  5. 不能随便插入空行
  6. WHERE子句要紧跟在FROM子句之后
  7. 单引号和双引号要看数据库语言，一般是单引号
  8. 所有包含 NULL 的计算，结果肯定是 NULL
  9. 字符串类型的数据原则上按照字典顺序进行排序，不能与数字的大小顺序混淆
  10. AND 运算符的优先级高于 OR 运算符。想要优先执行 OR 运算符时可以使用括号
  11. MAX/MIN 函数几乎适用于所有数据类型的列。SUM/AVG 函数只适用于数值类型的列
  12. 在聚合函数（count, sum, avg, max, min）的参数中使用 DISTINCT，可以删除重复数据
  13. SQL 子句的顺序不能改变，也不能互相替换（暂定：SELECT → 2. FROM → 3. WHERE → 4. GROUP BY）
  14. 数数用SELECT COUNT（输出数字）
  15. 使用 GROUP BY 子句时，SELECT 子句中不能出现聚合键之外的列名
  16. 在 GROUP BY 子句中不能使用 SELECT 子句中定义的别名
  17. GROUP BY 子句结果的显示顺序是随机的，除非自己定义
  18. 只有 SELECT 子句和 HAVING 子句(以及 ORDER BY 子句)中能够使用聚合函数

聚合与排序

聚合函数

将列的名字作为参数

  • COUNT(column):计算表中的记录数(行数)

  • SUM(column): 计算表中数值列中数据的合计值

  • AVG(column): 计算表中数值列中数据的平均值

  • MAX(column): 求出表中任意列中数据的最大值

  • MIN(column): 求出表中任意列中数据的最小值

使用聚合函数删除重复值(DISTINCT关键字) - 查询种类

分组

GROUP BY - 想要计算汇总结果（一般会使用 SELECT + 聚合函数 COUNT ）

执行结果

聚合键中包含 NULL 时，在结果中会以“不确定”行(空行)的形式表现出来。

WHERE + GROUP BY

GROUP BY和WHERE并用时SELECT语句的执行顺序：FROM → WHERE → GROUP BY → SELECT

GROUP BY常见错误

在 SELECT 子句中不能写了多余的列（MySQL以外的都不支持）

SELECT 子句中只能存在以下三种元素。

  • 常数 - ‘123’, ‘testing’
  • 聚合函数 - sum, avg, max, min
  • GROUP BY子句中指定的列名(也就是聚合键)
    以下有执行错误：
    SELECT product_name, purchase_price, COUNT(*) FROM Product
    GROUP BY purchase_price;
    因为用purchase_price作为聚合键，product_name会被合并，里面不同的数据无法在同一个purchase_price行里，所以报错。使用 GROUP BY 子句时，SELECT 子句中不能出现聚合键之外的列名。

    在GROUP BY子句中不能写列的别名

    SELECT product_type AS <font color=”red”pt, COUNT(*) FROM Product
    GROUP BY <font color=”red”pt;
    原因是在DBMS中，SQL语句的执行顺序不是代码顺序，而是FROM → WHERE → GROUP BY → SELECT。别名在SELECT中定义，然而GROUP BY先于SELECT执行，并不知道别名的定义，所以无法执行。（仅在PostgreSQL可以，但不是通用的）

    HAVING - 为聚合结果指定条件
